# Getting Started

Monerium provides a bridge between the traditional banking system and the blockchain, enabling the issuance of e-money (EURe) directly onto supported chains. Our services allow partners to integrate programmable payments, automate treasury management, and offer specialized financial products to their users.

## Sandbox vs Production environments

Monerium provides two environments, [**Sandbox**](https://sandbox.monerium.dev) and [**Production**](https://monerium.app).

### Sandbox

The Sandbox environment is a test environment that closely mirrors Production, but uses test blockchains and simulated funds (fake money).

It is intended for development and testing. Before creating new applications in Production, applications should be created in Sandbox and tested thoroughly.

To get EURe to your account on Sandbox, you can simulate incoming bank transfers [here](https://sandbox.monerium.dev/receive) after logging in to sandbox.monerium.dev.

Gas for test environments can be found here:

- [Sepolia faucet](https://sepolia-faucet.pk910.de/) to get sepETH
- [Chiado faucet](https://faucet.chiadochain.net/) to get test xDAI
- [Amoy faucet](https://faucet.polygon.technology/) or [Alchemy's faucet](https://www.alchemy.com/faucets/polygon-amoy) to get test Matic

### Production

The Production environment is used for live applications with real users, real funds, and real blockchains.

When the application has been fully tested in Sandbox and is ready to go live, the application is created and configured in the Production environment.

## Create application in Sandbox

To create an application in the Sandbox environment, you need a sandbox user account at [https://sandbox.monerium.dev](https://sandbox.monerium.dev). Once logged in, navigate to the developers section by clicking the profile icon in the top-right corner and select [**Developers**](https://sandbox.monerium.dev/developers/apps/create).

## Select your application kind

There are three kinds of applications you can create, each with a different use-case.

### Whitelabel

[Full Whitelabel guide →](./whitelabel)

- **Customer ownership.** You register users with their email (or a unique identifier) and receive a profile UUID to manage their entire lifecycle.
- **Full API integration.** Utilize the API to submit comprehensive KYC/KYB details, upload verification documents, and receive real-time status updates via webhooks.
- **Brand sovereignty.** Deliver a completely native experience where you issue dedicated IBANs and orchestrate payments under your own brand, ensuring users never need to interact with the Monerium interface.

### OAuth

[Full OAuth guide →](./oauth)

- **Rapid integration.** The fastest way to add Monerium functionality to any public DeFi or Web3 project.
- **Shared ecosystem.** Users can move their existing IBANs and balances between different apps in the Monerium community with explicit confirmation.
- **Path to growth.** Perfect for prototyping and proving a business case before upgrading to a Whitelabel solution.

### Private

[Full Private guide →](./private)

- **Direct control.** Designed for backend automation with no redirect URIs or external user management required.
- **Efficient treasury.** Allows companies to automate payroll or treasury movements at low latency without browser-based logins.
- **Simplicity.** No need to design onboarding flows or KYC interfaces; simply use your production credentials to manage your funds.

## Going live

To ensure a smooth transition from development to a live environment, every application must follow a specific "Going live" path. While all applications require a verified profile, the review process varies depending on the complexity of the integration.

| Requirement              | Whitelabel                                          | OAuth                                               | Private                              |
| :----------------------- | :-------------------------------------------------- | :-------------------------------------------------- | :----------------------------------- |
| **Profile verification** | Required (KYC/KYB)                                  | Required (KYC/KYB)                                  | Required (KYC/KYB)                   |
| **Sandbox testing**      | **Mandatory.** Must be tested with simulated funds. | **Mandatory.** Must be tested with simulated funds. | Optional but recommended.            |
| **Approval process**     | Manual approval process.                            | Manual approval process.                            | **Automatic** upon profile approval. |
